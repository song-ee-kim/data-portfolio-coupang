# Review Scraping & Sentiment Analysis (리뷰 수집 · 감성 분석)

이 프로젝트는 웹에서 상품 리뷰 데이터를 수집하고,  
이를 기반으로 **감성 분석 · 키워드 분석 · 토픽 모델링**을 수행하여  
상품 개선 인사이트를 도출하는 것을 목표로 합니다.

---

## 1. 프로젝트 목적

- 고객 리뷰에서 부정적인 감성의 주요 원인을 자동으로 파악
- 상품/카테고리별 감정 경향성 분석
- 커머스 운영(상품개선·카탈로그 개선)에 활용 가능한 인사이트 도출

---

## 2. 데이터 수집

### ✔ 수집 방식
- **웹스크래핑(Web Scraping)**: BeautifulSoup, Selenium  
- **API 기반 데이터 수집** (가능한 경우)

### ✔ 리뷰 데이터 예시 컬럼
- `review_text`
- `rating`
- `date`
- `product_name`
- `user_id` (익명화)
- `category`

---

## 3. 전처리 (Text Preprocessing)

- 한글 정제 (특수문자 제거, 소문자 변환 등)
- 토큰화(Mecab/Okt 등)
- 불용어 제거
- 형태소 기반 명사·형용사 중심 분석

---

## 4. 감성 분석 (Sentiment Analysis)

### ✔ 방법
- KNU 한국어 감성사전 기반 점수 부여
- 긍정/부정 비중 계산
- 상품별 평균 감성 점수 비교

### ✔ 시각화
- 긍·부정 리뷰 WordCloud
- 감성 점수 분포도
- 상위 부정 키워드 차트

---

## 5. 토픽 모델링 (Topic Modeling)

### ✔ 모델
- LDA(Latent Dirichlet Allocation)
- NMF(Non-negative Matrix Factorization) 비교 가능

### ✔ 산출물
- 주요 토픽별 상위 키워드
- 리뷰 주제 파악  
  예: “배송 지연”, “사이즈 불만”, “냄새”, “품질 문제”

---

## 6. 주요 인사이트 예시

- 배송 관련 부정 리뷰 비중 높음 (지연/포장 문제)
- 특정 카테고리(의류)는 **사이즈 오류** 관련 부정 리뷰 비중 ↑
- 제품 품질의 일관성 문제 제기 → 개선 방향 도출 가능

---

## 7. 향후 개선점

- KoBERT 기반 감성 분석으로 정확도 향상
- 카테고리별 전용 감성 사전 구축
- 토픽 모델링 결과를 상품 카탈로그 오류 탐지와 연결
